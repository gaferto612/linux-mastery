# Your capstone projects live here

Create a subfolder per project. Each subfolder should contain:

- A `README.md` explaining what the project is, what problem it solves, how to use it
- The actual code / configs / docs
- A `lessons-learned.md` at the end — what worked, what didn't, what you'd do differently

## Suggested layout

```
projects/
├── self-hosted-web-service/
│   ├── README.md
│   ├── nginx/
│   ├── systemd/
│   ├── scripts/
│   └── lessons-learned.md
├── health-reporter/
│   └── ...
└── ...
```

Pick projects from the parent README. Treat each one as if a stranger needs to follow your docs and reproduce your setup.
